Linux服务器 Claude Code安装教程

Claude Code是由Anthropic公司开发的一款AI驱动型的编程Agent命令行工具。这类产品也已经成为当代程序员代码开发不可或缺的重要工具。本文将介绍如何在 Linux 服务器上部署、配置个人Claude Code，利用CC-Switch绕过Claude Code本身的订阅需求，并且无需sudo 权限。

本文主要分成两部分：

  1. 先安装并验证 Claude Code
  2. 如果首次启动时遇到登录受限、地区限制，或者你本来就准备使用自定义 API provider，则继续安装并配置 cc-switch CLI

如果你只是想在服务器上快速把 Claude Code 跑起来，可以先看前半部分；如果你后续还需要切换 provider、配置 apikey、base_url、model，则重点看后半部分。

1 安装前准备

在正式安装之前，可以先确认当前 Ubuntu 环境里至少有 bash 和 curl。

如果这两个命令都能正常输出版本号，通常就可以继续后面的安装流程。

  2. 某些服务器环境下，可能会遇到登录流程不方便、浏览器认证不顺畅，或者当前网络/地区不支持直接使用的情况

如果你这里可以顺利登录，那么后续直接使用官方账号方式即可；如果你在这一步遇到问题，就可以继续配置 cc-switch CLI，改为使用你自己可用的 API provider。

5 什么是 cc-switch CLI

cc-switch CLI 可以理解为一个专门用来管理 AI 编码工具配置的命令行工具。它可以帮助你统一管理不同 provider 的配置，并支持为 Claude Code 切换不同的 API 设置。

如果你在 Claude Code 首次登录这一步卡住了，或者你本来就准备使用自定义 API key / base URL / model，那么 cc-switch CLI 会非常方便。

cc-switch CLI 官方仓库地址如下：

https://github.com/saladday/cc-switch-cli

它的 README 里给出了多种安装方式。对于没有 sudo 权限的 Ubuntu 服务器用户，推荐优先考虑下面两种思路：

  1. 直接使用官方安装脚本
  2. 手动下载二进制到自己的用户目录

10 小结

对于 没有 sudo 权限的 Ubuntu 服务器用户，安装 Claude Code 的思路并不复杂：

  1. 先用官方脚本安装 Claude Code
  2. 用 claude --version 验证是否安装成功
  3. 如果命令找不到，检查 ~/.local/bin 是否加入 PATH
  4. 如果首次登录不方便，或者你本来就要使用自定义 provider，则安装 cc-switch CLI
  5. 通过 provider add、provider switch、provider fetch-models 等命令完成配置

整体上，这套流程最大的关键并不是安装命令本身，而是两件事：

  • 用户目录下的命令是否已经进入 PATH
  • 当前 Claude Code 最终读取的是哪一套 provider / 环境变量配置

只要把这两个问题理顺，后面基本就可以稳定使用了。

如果在实际使用过程中还遇到其他问题，可以先自行查询相关资料进一步排查。本博客后续也会继续同步更多常见问题解答 FAQ，供大家参考。
