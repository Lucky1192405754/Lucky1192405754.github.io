理解 Transformer 的 Attention 机制

发表于2026-04-10|更新于2026-05-22|「CV」看得见的计算机视觉
|浏览量:

引言：为什么 Transformer 值得被“看见”

Transformer 已经成为现代 AI 的底层通用架构：大语言模型、视觉 Transformer、多模态模型，乃至许多序列建模系统，都在不同程度上继承了它的设计。

这一架构的难点不在于定义本身，而在于它很容易被压缩成若干公式、矩阵乘法和工程术语。读者通常知道 Attention(Q, K, V) 这条公式，也知道 Transformer 在实践中的重要性，但不一定容易建立一个稳定的结构认知：

  • token 是如何被切分并编码的；
  • 注意力矩阵的每一行、每一列分别表示什么；
  • 为什么要除以 $\sqrt{d_k}$；
  • 多头注意力带来的究竟是更多的图，还是更多的子空间；
  • 为什么训练阶段与推理阶段的瓶颈并不相同；
  • KV Cache、MQA/GQA、FlashAttention 这些优化分别作用在哪个层面。

本文的目标不是重复 Transformer 的标准定义，而是把这套结构重新放回图示、数值与公式之中，说明它如何从输入 token 出发，逐层建立上下文、压缩语义，并最终形成输出概率分布。

上图是原始 Transformer 论文中最具代表性的整体结构图。左侧是编码器堆栈，右侧是解码器堆栈。编码器中的每一层由 Multi-Head Self-Attention 和 Position-wise Feed-Forward Network 两部分组成；解码器则在此基础上额外插入一层 Masked Multi-Head Attention，用于保证自回归生成时当前位置不能访问未来 token。

这张图的意义在于为后文建立统一的结构坐标：

  • Input / Output Embedding 把离散 token 映射到连续向量空间；
  • Positional Encoding 为序列注入顺序信息；
  • Add & Norm 表示残差连接与归一化始终伴随各子层；
  • Multi-Head Attention 负责跨 token 的信息路由；
  • Feed Forward 负责逐位置的非线性变换；
  • 顶部的 Linear + Softmax 把最终隐藏状态投影回词表，得到输出概率分布。

后文对 QKV、缩放点积注意力、多头机制、归一化、FFN、生成与推理优化的讨论，都可以在这张总图中找到对应位置。

1. 为什么需要 Attention：从局部扫描到全局路由

在 Transformer 出现之前，主流序列模型大致沿着两条路线发展：

  • CNN：通过局部卷积核滑动扫描，逐层扩大感受野；
  • RNN / LSTM：沿时间步递归传播隐藏状态，把历史信息逐步带到当前位置。

这两类方法都很成功，但也都存在明确的结构性限制：

  1. 并行性不足：RNN 的时间步天然串行，长序列训练和推理都较慢；
  2. 长程依赖困难：CNN 需要堆叠较多层才能形成全局依赖；RNN 即便带有门控，也容易在超长序列中退化；
  3. 信息路由不够直接：传统网络更像“局部提取 + 逐层传递”，而不是“当前位置显式查询整段上下文”。

Attention 的核心突破在于：让当前位置的表示，直接对整段上下文进行一次可学习的全局寻址。

一个典型例子是下面这句话：

The door of the bank was broken, so he went to another bank to withdraw money.

当模型读到第二个 bank 时，其语义并不由局部邻域单独决定，而依赖整句上下文。Attention 的作用，就是允许当前 token 以连续向量的形式“回看”整段序列，并根据相关性重新分配上下文权重。

为了说明这一点，可以先看一个最小化的数值例子。设句子中只有 3 个词：

  • 小猫
  • 趴在
  • 窗台

当当前位置是 窗台 时，可以先给三个位置分配一组原始相关性分数：

  • 小猫: 1.2
  • 趴在: 2.6
  • 窗台: 1.8

把这组分数归一化之后，可以得到一组更接近概率的注意力权重，例如：

  • 小猫: 0.20
  • 趴在: 0.48
  • 窗台: 0.32

这还不是正式的 Transformer 公式，但已经足够说明注意力机制的基本任务：

对上下文中每个位置计算一个与当前任务相关的分数，再把这些分数转化为可用于信息聚合的权重。

在正式的 Transformer 中，这一过程会由向量表示、矩阵乘法与可训练参数来完成；后文出现的 Q、K、V、softmax、多头注意力以及后续的推理优化，本质上都围绕这件事展开。

2. Q、K、V：Transformer 如何提出问题、匹配线索、汇总信息

前一节的数值例子可以进一步形式化。对于任一当前位置，注意力计算实际上可以拆成三件事：

  1. 确定当前位置正在寻找什么；
  2. 判断上下文中哪些位置能够提供相关线索；
  3. 决定一旦关注这些位置，应当取回哪些内容表示。

Q、K、V 正是对这三件事的结构化表达。

2.1 Q、K、V 的语义角色

Transformer 中最核心的三个对象是 Query (Q)、Key (K) 与 Value (V)。

最常见的理解方式是信息检索：

  • Query：当前位置正在寻找什么信息；
  • Key：上下文中每个位置拥有什么可匹配的特征；
  • Value：该位置一旦被关注，最终会贡献什么内容表示。

在实现上，输入序列的隐藏表示 $X$ 会通过三组不同的线性映射，分别得到：

这一步的意义在于把同一个 token 分别投影到“提问”“索引”“内容”三种不同角色之中，使相关性判断与内容传递能够被解耦。

2.2 为什么必须独立投影

如果直接让原始向量 $X$ 与自身做点积，即 $XX^T$，那么相似度矩阵通常会偏向对称结构，自身位置也容易获得过大的权重。这样得到的注意力模式往往过于僵硬。

引入独立的 $W^Q, W^K, W^V$ 之后，同一个 token 就可以在不同语义子空间中承担不同功能：

  • 作为 Query，表示当前位置的需求；
  • 作为 Key，表示该位置可供匹配的特征；
  • 作为 Value，表示该位置可被聚合回来的内容。

这一设计使注意力不再只是“自相似比较”，而成为一种可学习的条件化匹配机制。

2.3 缩放点积注意力为什么要除以 $\sqrt{d_k}$

在正式写出公式之前，先看一个与 softmax 稳定性直接相关的事实：相关性分数不能无限增大。

例如：

  • 分数为 [1, 2, 3] 时，softmax 大致为 [0.09, 0.24, 0.67]；
  • 分数为 [10, 20, 30] 时，softmax 几乎会塌缩成 [0, 0, 1]。

后者的风险不在于排序错误，而在于分布过于尖锐，导致梯度迅速变小，训练中的可调节空间显著下降。

标准的缩放点积注意力写作：

其中缩放项 $\sqrt{d_k}$ 的作用，是把随着维度增长而不断变大的点积数值拉回到更稳定的范围内。如果 Q 和 K 各维度近似独立、均值为 0、方差为 1，那么点积 $q \cdot k$ 的方差会随 $d_k$ 线性增长；维度越大，softmax 越容易过早进入饱和区。

因此，除以 $\sqrt{d_k}$ 并不是公式中的装饰项，而是保持训练稳定性的必要条件之一。

2.4 两个常被忽略的实现细节：mask 与数值稳定性

Attention 的实现并不止于 softmax(QK^T)。至少有两个细节需要在结构层面明确区分。

1）Mask 不是只有一种

常见至少有两类 mask：

  • Padding mask：屏蔽补齐位置，避免模型把注意力分配到无意义 token；
  • Causal mask：在自回归模型中阻止当前位置访问未来 token，保证生成过程满足因果约束。

二者都作用在 score matrix 上，但语义完全不同。前者表示“这些位置无效”，后者表示“这些位置当前不可见”。

3. 从缩放点积到多头结构：Attention 在结构图中究竟对应什么

这张图对应最基本的 Scaled Dot-Product Attention。它把前一节的公式拆成了一个可逐步核对的计算流程：

  1. 先计算 $QK^T$，得到 Query 与所有 Key 之间的相关性分数；
  2. 再除以 $\sqrt{d_k}$，控制分数尺度；
  3. 如有需要，在 softmax 之前施加 mask；
  4. 经过 softmax 得到归一化权重；
  5. 最后与 $V$ 相乘，把权重加回到具体内容表示上。

这张图的价值在于把抽象公式变成清晰的算子链。许多实现细节——例如 mask 必须加在 softmax 之前、缩放项必须作用在 score 而不是输出上——都能在这张流程图中直接定位。

4. Multi-Head Attention：多个头不是“多张图”，而是多个语义子空间

单头 attention 已经能够完成一次全局加权，但这还不足以覆盖不同类型的相关性。以一句简单文本为例，某个位置同时可能需要处理：

  • 语法关系，例如动作主体；
  • 位置关系，例如动作的落点；
  • 语义关系，例如整句的主题对象。

如果所有类型的相关性都由一个 attention matrix 负责，模型需要在同一个相似度空间内同时兼顾多种标准。多头机制的作用，就是把这些不同标准拆分到多个子空间内并行处理。

4.1 多头的真正作用

这张图展示了多头结构的关键变化：输入不会只经过一次 Attention，而是先分别经过多组线性映射，形成多组头部的 $Q_i, K_i, V_i$，随后每个 head 独立完成一次 Scaled Dot-Product Attention，最后把各头输出 concat 起来，并通过 $W^O$ 映射回原始模型维度。

图中最关键的三点如下：

  1. Q、K、V 的线性映射是按 head 分开的，这使不同头可以学习不同的相似性标准；
  2. 中间的 attention 模块是并行执行的，因此多头机制不是重复同一张图，而是在多个子空间中同时做信息路由；
  3. Concat + Linear 是必要步骤，各头输出必须重新融合回统一表示空间。

更准确的说法是：

多头注意力把输入表示投影到多个低维子空间中，并在这些子空间里并行执行不同的信息路由。

5. Transformer Block 里除了 Attention 还有什么

Attention 解决的是 token 与 token 之间如何交换信息，但完整的 Transformer block 还必须同时处理另外几个问题：

  • 新旧信息如何稳定叠加；
  • 表示尺度如何保持稳定；
  • 位置间完成路由之后，单个 token 如何进一步进行非线性特征变换。

因此，一个完整的 Transformer block 至少还包括三个关键部件：

  1. Residual connection：保证深层网络中的梯度与信息通路稳定；
  2. Normalization：控制隐藏状态尺度，降低训练不稳定性；
  3. FFN / MLP：在每个 token 内部完成更强的非线性特征重编码。

5.1 LayerNorm、Pre-Norm 与 RMSNorm

这张图左侧对比了 BatchNorm 与 LayerNorm 的归一化维度。BatchNorm 会跨 batch 统计均值和方差，因此更依赖批量维度上的稳定分布；LayerNorm 则在单个样本的特征维度内部完成归一化，因此更适合序列模型与变长输入。

对 Transformer 而言，LayerNorm 的作用主要体现在两点：

  • 不依赖 batch 维度上的统计一致性，更适合自回归建模；
  • 把每层输入输出的数值尺度控制在较稳定的范围内，降低深层堆叠带来的训练不稳定性。

右侧公式给出了标准归一化过程：先减去均值，再除以标准差，最后乘以可学习的缩放参数并加上偏置。后续 RMSNorm 的变化，本质上就是对这套机制的进一步简化。

6. 从论文到工程：实现层面真正关键的细节

Transformer 的结构图相对清晰，但从论文到工程实现，中间仍然存在大量对训练稳定性与推理效率有决定性影响的细节。

6.1 位置编码与顺序信息

这张图左侧展示了正余弦位置编码在不同位置与不同维度上的数值分布，右侧给出了对应公式。图像中从低频到高频的周期变化说明：不同维度会以不同波长编码位置信息，因此模型既能感知相邻位置，也能在更长尺度上保留相对顺序关系。

这张图的关键结论有两点：

  • 不同位置会得到可区分的表示；
  • 相对位移可以在后续线性变换与点积中保留可利用的模式。

Self-Attention 对输入序列本身并不敏感于顺序。没有位置信息，它看到的更像一个无序 token 集合。

传统 Transformer 使用固定正余弦位置编码，而现代大模型更常见的是 RoPE（Rotary Position Embedding）。RoPE 的价值在于：

  • 位置信息直接体现在 Q/K 的相对旋转关系中；
  • 更适合长序列扩展；
  • 在当代 Decoder-only 模型中几乎已经成为事实标准。

6.3 训练和推理并不是同一条路径

训练时：

  • 整段序列可以并行送入模型；
  • causal mask 保证不访问未来位置；
  • 核心瓶颈通常是吞吐、显存与数值稳定性。

推理时：

  • 生成过程逐 token 进行；
  • 每一步只新增一个 token；
  • 核心瓶颈不再只是 FLOPs，而是重复计算与 memory movement。

这一区分直接决定了后文 KV Cache、GQA 与 FlashAttention 的工程意义。

6.4 softmax 之前与之后，哪些操作不能交换顺序

以 mask 为例，正确顺序是：

  1. 先计算 score；
  2. 再把非法位置填成极小值；
  3. 然后做 softmax；
  4. 最后再与 $V$ 聚合。

如果先 softmax 再遮掉某些位置，就会破坏概率归一化语义，最终改变模型行为。

7. 自回归生成：Transformer 如何真正“生成”下一个 token

对于 Decoder-only 模型而言，生成过程可以直接在总结构图的右半部分理解：模型在每一步接收已经生成的前缀，经过 masked self-attention、FFN 以及残差归一化之后，取最后一个位置的隐藏状态，再通过线性层与 softmax 得到下一个 token 的概率分布。

其计算链路可概括为：

  1. 对输入序列进行 tokenization；
  2. 经 embedding + positional encoding 得到初始表示；
  3. 多层 block 通过 attention 与 MLP 逐层重写表示；
  4. 取最后一个位置的 hidden state；
  5. 乘以 LM head 得到 logits；
  6. 经 temperature / top-k / top-p 等策略采样下一个 token。

“预测下一个词”这一表述并没有问题，但真正关键的是：

这个下一个 token 的概率分布，凝结了前面所有层中对上下文的压缩、对齐、筛选与重路由。

8. KV Cache、MQA/GQA、FlashAttention：现代推理优化到底在优化什么

当模型规模和上下文长度持续增长时，系统瓶颈已经不只来自数学计算本身，还来自重复计算、显存占用与带宽成本。

8.1 KV Cache：避免把历史一遍又一遍重算

在自回归推理中，每一步只新增一个 token，但如果每次都重新计算全部历史 token 的 K/V，就会产生大量重复开销。

KV Cache 的核心思想是：

  • 历史 token 的 K/V 一旦算出，就缓存起来；
  • 新 token 只需计算自己的 Q（以及自己的 K/V）；
  • 然后用新的 Q 与缓存中的 K/V 做注意力计算。

它并不改变模型语义，只是在推理阶段复用了历史状态，从而消除了大量重复计算。

8.2 MHA、MQA、GQA 的本质是“KV 的共享程度不同”

标准多头注意力（MHA）中，每个 head 都有独立的 K/V，表达能力强，但 KV Cache 占用也更大。

于是出现了：

  • MQA（Multi-Query Attention）：多个 query head 共享同一组 K/V；
  • GQA（Grouped-Query Attention）：在若干组 query head 内共享 K/V，是 MHA 与 MQA 之间的折中。

它们解决的并不是训练是否收敛，而是长上下文推理时 KV Cache 的显存与带宽压力。

8.3 FlashAttention：优化的不是公式，而是内存 IO

FlashAttention 容易被误解为一种 attention 近似算法。更准确的说法是，它首先是一种 IO-aware exact attention kernel。

它所做的不是改变 Attention 的数学定义，而是：

  • 通过 block / tile 方式重排计算；
  • 减少 HBM 与 SRAM 之间的来回搬运；
  • 让 softmax 与矩阵块计算在更局部的范围内完成；
  • 在不改变最终结果的前提下，大幅降低显存访问成本。

因此，FlashAttention 的关键贡献不在于新公式，而在于把 attention 的瓶颈从“理论算子数量”转移到“实际硬件上的内存访问方式”上来理解。

9. 从 NLP 到 ViT、CLIP 与多模态模型：Transformer 为什么能扩展得这么远

Transformer 的可扩展性并不来自某一种特定模态，而来自它对 token 化表示 + 统一路由机制 的兼容性。文本、图像乃至多模态输入，只要能够被改写成离散 token 或连续向量序列，就能够进入相近的结构框架。

9.1 ViT：把图像重新表述为 patch 序列

这张图说明了 ViT 与标准 Transformer 的对应关系。输入图像首先被切分为固定大小的 patch，每个 patch 经过线性投影后变成一个 token；随后这些视觉 token 与一个额外的分类 token 一同送入 Transformer Encoder。

图中最关键的观察点有三个：

  • patch embedding 取代了传统 CNN 的卷积特征提取起点；
  • 位置编码依旧存在，说明视觉 token 同样需要空间位置信号；
  • 分类结果来自全局聚合后的表示，而不是单个局部卷积响应。

因此，ViT 的关键不在于“发明了一种新的视觉 attention”，而在于把二维图像重写成一维 token 序列，再复用 Transformer 编码器结构。

9.2 Cross-Attention：从“看自己”走向“去别的模态里查信息”

这张图展示了跨模态建模中一个核心思想：文本与图像可以先各自编码成向量表示，再通过共享语义空间建立匹配与对齐关系。虽然 CLIP 本身并不等同于典型的 encoder-decoder cross-attention，但它清楚说明了不同模态最终都可以被组织成 token 或向量集合，并在统一表示空间中进行比较。

在更典型的 cross-attention 结构中，目标序列当前位置的 Query 来自当前模态，而 Key / Value 来自另一模态或另一序列。这样一来，当前位置的表示就不再只从自身上下文取证，而是可以从外部表示集合中检索与当前任务最相关的信息。

因此，从文本到视觉，再到跨模态系统，Transformer 的可迁移性主要来自“统一表示 + 统一路由”的范式，而不是某个单一任务技巧。

10. 现代 Transformer 家族：哪些变化是主线，哪些只是局部工程优化

近些年的模型演化大致可以概括为以下几条主线。

10.1 结构范式

  • Encoder-only：BERT 风格，擅长双向表征与判别任务；
  • Decoder-only：GPT / LLaMA 风格，擅长自回归生成；
  • Encoder-Decoder：T5、原始机器翻译 Transformer 风格，适合序列到序列映射。

10.2 位置与归一化

  • sinusoidal positional encoding → RoPE
  • LayerNorm → Pre-Norm → RMSNorm

10.3 推理效率

  • MHA → MQA / GQA
  • 普通 attention kernel → FlashAttention / fused attention
  • 无 cache → KV Cache

10.4 模态扩展

  • 文本序列 → patch 序列（ViT）
  • 文本 + 图像双编码（CLIP）
  • 视觉编码器 + LLM 投影（LLaVA / 早期 Qwen-VL）
  • 更原生的统一多模态 token 系统（如 GPT-4o、Gemini 等路线）

这些变化中，有的属于模型表征能力与结构范式的演进，有的属于在更长上下文、更大参数量与更严格延迟约束下维持可部署性的系统优化。二者共同构成现代 Transformer 的主线。

11. 结语：Transformer 不是一张热力图，而是一套统一的信息路由哲学

全文的核心可以归结为以下几点：

  • Q、K、V 不是抽象符号，而是“提问—匹配—取回信息”的计算结构；
  • 缩放点积注意力的关键不只在公式本身，还在于尺度控制、mask 位置与数值稳定性；
  • Multi-Head Attention 不是简单复制，而是在多个子空间中并行执行信息路由；
  • Transformer block 不只有 attention，还依赖残差、归一化与 FFN 共同维持稳定训练与强表示能力；
  • 训练与推理不是同一条路径，因此 KV Cache、GQA、FlashAttention 等优化在系统层面必不可少；
  • 从文本到视觉再到多模态，Transformer 的持续扩展依赖的是统一 token 化与统一路由范式。

因此，Transformer 的本质并不在某一张 attention 图上，而在于它提供了一套统一的机制，用于学习“谁应当向谁传递什么信息”。
