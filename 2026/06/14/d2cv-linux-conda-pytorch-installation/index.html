安装

在正式进入计算机视觉的模型、数据和实验之前，首先需要准备一个稳定、可复现的运行环境。环境一旦混乱，原本属于模型、代码或数据的问题，就容易被误判为算法本身的问题，后续排查也会随之失去秩序。

本文将给出在 Linux 环境下完成 Conda 环境与 PyTorch 环境的安装教程，并建立一个可以继续用于后续实验的基础环境。

一点实践提醒

Linux 环境配置里，真正容易出问题的地方通常不在安装命令本身，而在“环境是否干净”和“版本是否混装”。后续实验中，建议保持以下习惯：

  • 每次开始工作前，先确认自己是否已经 conda activate pytorch；
  • 不要在同一个环境里反复混装来源不明的包；
  • 如果某次安装之后环境明显混乱，优先考虑新建环境，而不是在旧环境里无限修补；
  • 当安装命令与教程不一致时，优先参考当前官方页面与报错信息，而不是死记旧版本命令。

这些习惯看起来只是工程细节，但它们直接影响后面做实验时的稳定性。环境越清楚，模型问题和代码问题就越容易被看清。

小结

到这里为止，Linux 环境下最基础的一组准备工作已经完成：安装 Miniconda，创建独立的 Conda 环境，并在其中安装 PyTorch。后面的文章如果需要运行张量操作、图像读写、简单网络结构或者更完整的视觉实验，都可以在这个基础上继续推进。

只要 pytorch 环境中已经能够成功导入 torch，这一节的目标就已经达到。接下来更值得关心的问题，将转向这些张量、这些图像以及这些模型本身的结构与作用。
