计算机视觉简介

时至今日，人们日常接触到的许多智能系统，已经不再满足于处理结构化表格、字符串或明确规则，而是需要直接面对图像、视频和更复杂的视觉信号。一个购物平台可以自动识别用户上传的商品图片，一个城市交通系统可以根据监控视频分析车流与行人状态，一个医学系统可以依据影像辅助筛查病灶，一个机器人也需要借助视觉感知来理解环境、定位目标并完成动作规划。对于这些问题，仅仅依赖人工预先写定规则，往往很难得到足够可靠的解决方案。

这正是计算机视觉存在的根本背景。它所关心的，并不是如何把“看见”简单地翻译成几个接口调用，而是如何让计算机从图像中形成对目标、区域、结构、关系和场景的有效表征。表面上看，人眼对图像的理解似乎轻而易举：我们几乎不会费力去判断一张照片里有没有人、桌子在哪个位置、天空与建筑的边界又在哪里。但一旦试图把这一过程明确地写成规则，就会发现其中牵涉的变化远比直觉复杂。光照会变化，视角会变化，尺度会变化，遮挡会出现，背景也可能异常混乱；同一个物体在不同图像中的像素排列完全不同，而完全不同的物体有时又会在局部外观上呈现惊人的相似性。

因此，计算机视觉的核心难点并不只是“图像很多”，而是图像中的语义与像素之间并不存在一条容易手工写出的映射路径。我们知道一只猫、一辆汽车或一张人脸在视觉上意味着什么，但若要求程序员仅靠显式规则为每一种对象写出完整判据，这几乎是不现实的。也正因为如此，现代计算机视觉越来越依赖数据驱动的方法：与其为每一种情况手工枚举规则，不如构造一类具有足够表达能力的模型，再通过数据去学习从输入图像到目标输出之间的映射关系。

这一章的目标，是对计算机视觉作一个整体性的引入说明。后面的章节会分别讨论图像表示、张量结构、卷积、特征提取、分类、检测、分割以及更现代的视觉模型，但在正式展开之前，先从整体上理解计算机视觉要解决什么问题、依赖哪些基本组件、为什么它会成为现代人工智能的重要方向，仍然是必要的。

日常生活中的计算机视觉

计算机视觉早已渗透到日常生活之中。手机解锁时的人脸识别、相册中的人物聚类、地图应用中的街景理解、电商平台的以图搜图、短视频平台的内容审核、车载系统中的车道检测与行人识别，这些任务都在不同程度上依赖视觉模型。很多时候，用户只会感受到一个“功能”，而不会意识到其背后实际上是图像采集、特征表示、模型推理和结果决策的一整套流程。

设想一个简单的例子：如果希望程序自动识别一张图片中是否存在猫，那么最直接的困难并不在于“猫”这个概念无法定义，而在于它在图像中的外观变化太大。猫可能面对镜头，也可能背对镜头；可能只占据画面的一小部分，也可能充满整个画面；可能处于自然光下，也可能处于夜间室内环境；甚至一只猫的一部分轮廓，也可能被沙发、窗帘或另一个物体遮住。仅凭手工规则很难穷举这些变化。

对于这类问题，一个更可行的思路是收集大量带标签的图像样本，构造一类可学习的模型，并通过训练逐步调整模型参数，使其在给定图像时能够输出合理的判断结果。这里的“合理”，并不意味着模型像人一样理解了猫的概念，而是意味着它在统计意义上学会了怎样把图像中的模式与标签关联起来。换言之，计算机视觉的大量任务，本质上都可以视作从视觉信号到某种结构化输出的映射学习问题。

计算机视觉中的关键组件

无论视觉任务的具体形式如何变化，现代计算机视觉系统通常都离不开几个基本组件：数据、模型、目标函数以及优化算法。它们共同构成了后续大多数章节的技术基础。

数据

如果没有图像数据，计算机视觉几乎无从谈起。每一张图像都可以被视作一个样本，它的最原始表示是像素值的排列；若进一步扩展到视频，则样本还会携带时间维度。在分类任务中，标签可能是一个类别；在检测任务中，标签通常包含类别和边界框；在分割任务中，标签则可能细化到每个像素所属的语义类别。数据的组织方式不同，决定了任务的形式、模型的输出结构以及训练目标的定义方式。

处理视觉数据时，一个基本事实是：图像虽然看起来是“自然”的，但进入模型之前必须经过数值化表示。以一张 $224\times224$ 的 RGB 图像为例，它常常被表示为属于 $\mathbb{R}^{3\times224\times224}$ 的张量。若按批量处理，则可以进一步写成 $\mathbb{R}^{B\times C\times H\times W}$。这种表示方式会贯穿后续所有视觉模型，因为卷积、归一化、池化和注意力机制最终操作的对象，都是这些高维数组。

仅有大量数据并不自动意味着性能良好。数据是否具有代表性、标注是否准确、类别是否均衡、采集分布是否与测试场景一致，这些问题同样决定模型能否在真实环境中可靠工作。计算机视觉中的不少失败案例，并不是模型结构本身出了问题，而是训练数据没有覆盖足够多的变化形式，或者标注标准本身不一致。

模型

计算机视觉中的模型，本质上是从图像表示到输出结果的变换系统。这个输出可以是类别、边界框、像素标签、特征向量，也可以是生成后的图像、字幕或三维结构。传统视觉方法往往把“特征提取”和“决策模型”分开处理，例如先设计边缘、角点、局部描述子，再把这些特征送入分类器或回归器。深度学习兴起之后，越来越多的视觉系统改为直接从原始图像出发，通过多层可学习变换自动形成特征，再完成最终任务。

从这一角度看，计算机视觉模型的一个关键能力就在于表示学习。靠近输入的层通常更接近纹理、边缘和局部模式；更深层的表示则逐渐聚合为部件、对象乃至场景层面的结构信息。这样的层级化表示，并不等同于人类视觉系统的精确模拟，但它确实提供了一种有效的工程路径，使得模型能够在大规模数据上自动学习有用的视觉特征。

目标函数

模型需要一个明确的优化目标。没有目标函数，参数更新就失去了方向。视觉任务中的目标函数会随任务形式而变化：分类任务常见的是交叉熵损失，检测任务往往包含分类损失和边界框回归损失，分割任务则常常结合像素级交叉熵、Dice 损失或 IoU 相关目标。生成任务、对比学习和自监督学习又会使用完全不同的目标形式。

目标函数的作用，不只是判断“结果对不对”，还决定模型将优先学习什么样的行为。举例来说，一个检测模型如果只优化分类准确率，而忽略定位质量，那么它可能知道图中有猫，却很难在图中准确框出猫的位置；反过来，如果定位目标定义得过于严格，模型又可能在训练早期难以稳定收敛。因此，视觉任务中的目标函数设计本身就是模型设计的重要组成部分。

优化算法

当数据、模型和目标函数都确定之后，还需要一种方法去寻找较优参数。深度学习视觉模型中最常见的优化方法仍然是梯度下降及其变体，例如 SGD、Momentum、Adam 或 AdamW。它们通过反向传播计算梯度，再沿着能够降低损失的方向更新参数。

在计算机视觉里，优化问题往往比表面看起来更复杂。一方面，模型参数规模大，训练过程长，学习率、批量大小、权重衰减和归一化策略都会影响训练稳定性；另一方面，视觉任务本身通常伴随着高分辨率输入、多分支结构和大规模特征图，这使得显存、速度与收敛性之间经常需要折中。因此，“如何训练模型”在视觉领域从来不是附属问题，而是系统性能的一部分。

各种计算机视觉问题

如果把计算机视觉只理解成图像分类，那么这个领域会显得过于狭窄。事实上，视觉任务的类型远比“判断图里是什么”丰富。不同任务的输入形式、输出结构以及评价指标都不相同，后续模型设计也因此出现了明显分化。

图像分类

图像分类是最基础的视觉任务之一。给定一张图像，模型输出其所属类别。例如，输入一张动物图片，模型需要判断它属于猫、狗、鸟或其他类别。在这个任务里，图像整体被看作一个样本，输出通常是类别概率分布或最终类别标签。

分类任务的重要性并不只在于它本身有广泛应用，更在于它为后续视觉模型的发展提供了一个相对清晰的基准环境。许多经典卷积网络最初都是在图像分类基准上发展起来的，它们学到的特征也经常被迁移到检测、分割与检索等任务之中。

目标检测

分类只能回答“是什么”，却无法回答“在哪里”。目标检测进一步要求模型在图像中定位出一个或多个对象，并给出每个对象的类别与边界框。与分类相比，检测的难点在于同一张图像可能同时包含多个目标，而且这些目标可能大小不同、位置不同、相互遮挡，甚至处于复杂背景之中。

检测任务的出现，使视觉模型不再只输出一个全局标签，而必须显式处理位置与尺度信息。这也是为什么边界框、锚框、多尺度特征和非极大值抑制会成为检测模型中的核心概念。

语义分割与实例分割

在检测任务中，模型通常只给出矩形框；而在分割任务中，模型需要更精细地理解图像。语义分割要求对每个像素赋予语义类别，例如道路、天空、行人或车辆；实例分割则进一步区分同类别的不同个体，例如图中出现的每一辆车、每一个人。

分割任务的重要性在于，它逼迫模型学习更细粒度的空间结构和边界信息。对于自动驾驶、医学影像分析、遥感解译和机器人感知而言，这类像素级理解往往比单纯分类更接近真实需求。

关键点检测、匹配与三维视觉

除了分类、检测和分割，计算机视觉还包括关键点检测、姿态估计、局部特征匹配、三维重建、立体视觉和视觉 SLAM 等方向。此时，模型的输出可能不再是类别，而是若干结构化坐标、对应关系或三维几何参数。例如，在人体姿态估计中，模型需要给出关节位置；在三维重建中，模型则试图从一组二维视图恢复三维结构。

这些任务共同说明了一点：计算机视觉并不只是“识别物体”，而是围绕视觉信息建立表征、推断结构并支持决策的一个完整领域。

监督学习、无监督学习与表示学习中的视觉问题

与机器学习其他分支一样，计算机视觉中的任务也可以放到监督学习、无监督学习和与环境交互的更大框架中理解。

在监督学习中，图像通常带有明确标签，例如类别、框、掩码或关键点位置。大多数工业级视觉系统都建立在监督学习基础上，因为它们的目标清晰，评价标准也较为明确。分类、检测与分割的大量成功案例，都来自这一范式。

在无监督学习和自监督学习中，问题会变得不同。此时不再依赖昂贵的人工标签，而是试图从图像本身的结构中学习有用表示。例如，模型可以通过预测图像局部、恢复缺失区域、对比不同视角或不同增强版本来形成表征。这类方法的重要性越来越高，因为视觉数据规模巨大，而高质量标注往往昂贵且耗时。

表示学习在视觉领域尤其关键。一个模型之所以能够迁移到多个任务，不在于它记住了某个数据集的答案，而在于它学到了一种更稳健的中间表示。后续无论是做分类、检测还是检索，真正可迁移的往往都是这些表示本身。

计算机视觉的起源与发展

计算机视觉并不是随着深度学习突然出现的。在深度神经网络成为主流之前，视觉领域长期积累了大量经典方法，包括边缘检测、角点检测、纹理描述、几何变换、立体匹配、图像分割和局部特征表示。很长一段时间里，视觉研究的核心路径是：由研究者手工设计特征，再用相对浅层的模型完成分类或匹配。

这种方法并非无效。Canny 边缘检测器、SIFT、HOG、光流估计和图割等经典技术，至今仍然是理解视觉任务的重要材料。它们直接塑造了人们对图像结构、几何关系和局部不变性的理解，也为后来的深度模型提供了许多问题意识。

真正的转折点，出现在大规模数据、GPU 计算和深度神经网络结合之后。卷积神经网络在图像分类基准上的突破，改变了视觉领域的技术重心：特征不再主要依靠人工设计，而是通过端到端训练自动学习得到。随后，检测、分割、视频理解、视觉生成和多模态视觉模型迅速发展，计算机视觉由此进入了一个新的阶段。

计算机视觉的成功案例

现代计算机视觉的成功并不局限于学术指标。智能手机中的人脸解锁、工业质检中的缺陷识别、医学影像中的病灶检测、安防系统中的行人分析、自动驾驶中的环境感知、遥感图像中的目标提取、零售系统中的商品识别，都是这一领域已经产生显著影响的例子。

视觉模型之所以具有如此广泛的应用价值，是因为图像本身承载了极丰富的信息，而很多现实任务最终都可以被转化为对这些信息的检测、分割、排序、匹配、预测或生成。与文本和表格相比，视觉数据的结构更复杂，但这也意味着一旦模型能够建立有效表征，它所能支撑的应用范围也会更广。

计算机视觉的特点

如果要概括现代计算机视觉的若干关键特征，那么至少包括以下几点。

第一，视觉数据天然具有高维结构。图像并不是一个普通的长向量，而是带有局部邻域、通道关系、尺度变化和空间拓扑的结构化对象。模型若忽略这些结构，通常很难高效学习。

第二，表示学习在视觉领域具有核心地位。视觉系统能否成功，往往取决于它是否学到了足够稳健的中间特征，而不只是最后一个分类器是否足够复杂。

第三，端到端训练深刻改变了视觉方法的组织方式。过去由人工分拆的特征提取、局部描述、几何匹配和决策过程，越来越多地被统一到一个可学习系统中。

第四，视觉模型与计算资源密切相关。输入分辨率、特征图大小、模型深度、训练批量和注意力范围都会直接影响计算开销，因此视觉研究从一开始就伴随着明显的系统约束。

最后，计算机视觉仍然是一个持续变化的领域。卷积网络并没有消失，Transformer 也没有终结演化；检测、分割、生成和多模态方法仍在不断重构人们对视觉问题的理解方式。对于学习者而言，更重要的并不是记住某一代模型的名称，而是逐步理解这些模型分别在解决什么问题、利用了什么结构、又引入了哪些新的代价。

配图建议

本章如果要配图，建议优先采用三类图，分别承担不同的说明任务。

第一类是“典型视觉任务示意图”，适合手绘或用 Python 简单排版生成。可以在同一张画布中并列展示分类、检测、分割和关键点估计四种输出形式，让读者直接看到同一幅图像在不同任务下对应的结果差异。这类图更适合自己制作，因为结构清楚、版权风险低，也便于统一整章风格。

第二类是“经典数据集或经典任务样例图”，适合从公开数据集或经典论文中选取具有代表性的图像片段。例如可以选用 ImageNet 风格的分类图、COCO 风格的检测图、Cityscapes 风格的分割图。但若采用这一路线，最好只引用公开可用的示意样例，并在文末统一说明来源，而不要在正文里频繁插入出处说明。

第三类是“视觉处理流程图”，适合用 Python、PPT 或矢量工具手绘。内容可以是“图像输入 → 特征提取 → 任务输出”的基本链条，也可以是“像素 → 边缘/纹理 → 部件 → 对象 → 场景”的分层表示示意图。这类图最容易和后续章节衔接，也最符合 D2CV 系列“看得见”的定位。

如果只选一种，我建议优先自己绘制“同一输入对应多种任务输出”的总览图。这张图在后续检测、分割、关键点和视频理解章节中都可以反复复用，具有较高的章节间连接价值。

小结

计算机视觉研究的是如何让计算机从图像或视频中形成有效表征，并完成识别、定位、分割、匹配、重建与生成等任务。它的发展并不只是模型规模的扩张，而是数据、表示、目标函数、优化算法和计算资源共同作用的结果。对后续章节而言，最重要的不是立刻掌握所有任务细节，而是先建立一个整体视角：视觉问题为什么困难，视觉模型依赖哪些基本组件，不同任务之间又是如何分化出来的。有了这一层背景，后面关于图像表示、卷积、特征图与视觉模型的讨论，才会更容易落到实处。
